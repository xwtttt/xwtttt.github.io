方法

方法定义

注意事项

  • 访问修饰符：可以省略，默认 private ，可以根据需要定义成 public
  • 方法名：一般是“动词”或者“动宾短语”，采用 Pascal ，首字母大写
  • 参数列表：根据需要添加，也可以省略。参数列表要求以 类型 参数名 的形式，使用逗号分隔，首字母一般不大写
  • 有返回值：使用 return 返回，return语句后面不能再有其他语句
  • 没有返回值：如果不反回任何数据，使用 void 表示

方法调用

调用规范

通过对象名.方法名（参数1，参数2，……）进行调用

变量的分类及其作用域

  • 在方法的内部定义的变量叫做“局部变量”，只能在该方法内部使用
  • 在方法外部，类的内部定义的变量，称为“成员变量”，也叫“字段”，可以在类的内部或者外部使用（很少在外部使用）

方法重载

当一个类中有重名的方法，但是方法的参数个数或者类型不同时，会根据不同的参数个数或者类型动态地调用不同的方法

方法重载的好处

  1. 减少类的对外接口（只显示一个方法），降低类的复杂度
  2. 便于用户使用（相同功能的方法名称一样）和识别

    方法重载的条件

  3. 方法的名称必须一样
  4. 方法的参数个数或者类型不一样
  5. 方法重载与返回值无关

  • 静态方法的调用：类名.方法名

静态成员使用经验

  • 静态成员在程序运行时被调入内存中，并且在系统未关闭之前不会被回收
  • 类的成员使用非常频繁时，可以考虑使用static修饰，但是不要使用过多
  • 静态成员不能直接调用实例成员（静态方法不能直接调用实例方法）

    构造方法

    构造方法在使用时要求和类名一致
    当没有写构造方法的时候，会自动生成无参构造方法

    有参数的构造方法

  • 参数的类型和顺序同样需要和定义一致
  • 有参数的构造方法可以让用户轻松选择使用何种方式完成对象的初始化工作
  • 使用有参数的构造方法能有效避免单个初始化对象属性的麻烦
  • 如果相约束用户创建对象时必须完成某些属性的初始化工作，则可以去掉无参构造方法

构造方法和实例方法

构造方法

  • 用于对象的初始化，一个类中至少有一个构造方法
  • 不能显示调用，只能在创建对象时，使用new来调用
  • 构造方法不能有返回值
  • 构造方法必须和类名一致

    实例方法

  • 用于表示对象能够干什么，一个类中可以没有任何实例方法
  • 只能显示调用对象名.方法名
  • 可以有返回值，没有时必须使用void表示
  • 方法命名要有意义，一般是动词+名词的形式，不能与类名相同，命名规范通常使用 Pascal 命名法

    this关键字

  • 当成员变量和局部变量重名时使用this区分
  • this表示当前类的对象，用于访问该类的成员变量或者方法
